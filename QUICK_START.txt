✅ CORE RECONCILIATION ENGINE
   • Automatic payout matching by amount
   • Multi-file reconciliation (Balance + Payouts + Bank)
   • Net activity calculation (Charges + Refunds + Fees + Disputes)
   • Missing payout identification
   • Exception detection and flagging
   • Amount mismatch identification

✅ CLIENT ISOLATION SYSTEM
   • Slug-based unique URLs (/portal/client_a, /portal/client_b, etc.)
   • Independent sessions per client
   • No cross-client data leakage
   • Automatic session cleanup after 1 hour

✅ FILE PROCESSING
   • Drag-and-drop CSV upload interface
   • Whitespace stripping and data cleaning
   • Case-insensitive matching
   • Real-time validation and feedback
   • In-memory processing (zero data persistence)
   • Automatic file cleanup after report generation

✅ PROFESSIONAL EXCEL REPORTS
   • Executive Summary tab (totals + summaries)
   • Payout Matching tab (detailed transaction list)
   • Exceptions tab (flagged issues + severity)
   • Audit Log tab (operation tracking)
   • Professional formatting with colors & borders
   • Currency formatting and calculations

✅ SECURITY & PRIVACY
   • User authentication backed by Flask-Login
   • Passwords hashed with Werkzeug (generate/check)
   • Each user tied to a single client slug (403 on cross‑slug access)
   • Session cookies HttpOnly/secure with environment toggle
   • Input validation on all uploads
   • No persistent client data on server
   • Security headers (nosniff, XFrame, XSS protection)
   • CSRF protection framework via Flask-WTF
   • Error logging without data exposure

💡 HOW IT WORKS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Step 1: UPLOAD
   • Client selects their unique portal (/portal/client_slug)
   • Uploads three CSV files via drag-and-drop
   • System validates and displays preview

Step 2: MATCH
   • "The Matchmaker" algorithm matches payouts to bank deposits by amount
   • Identifies matching transactions
   • Flags mismatches and missing items

Step 3: AUDIT
   • "The Auditor" calculates net activity
   • Detects exceptions and discrepancies
   • Compiles detailed exception list

Step 4: REPORT
   • Generates professional Excel file (reconciliation_report.xlsx)
   • 4 tabs: Executive Summary, Payout Matching, Exceptions, Audit Log
   • Download automatically triggered
   • Client data deleted from server


🔒 SECURITY ARCHITECTURE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

✓ IN-MEMORY PROCESSING
  • Files never stored on disk
  • Processed entirely in RAM
  • Deleted immediately after report generation

✓ CLIENT ISOLATION
  • Each client gets unique URL slug
  • Sessions validated on every request
  • No cross-client data access possible

✓ SESSION MANAGEMENT
  • Active reconciliators stored in memory dictionary
  • Auto-cleanup after 1 hour inactivity
  • Single client per session

✓ SECURITY HEADERS
  • X-Content-Type-Options: nosniff
  • X-Frame-Options: DENY  
  • X-XSS-Protection: 1; mode=block

✓ INPUT VALIDATION
  • CSV file validation
  • File type checking
  • Size limits (50MB max)
  • Type casting on numeric fields

📋 API ENDPOINTS REFERENCE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

GET /
   Home page with client selection

GET /portal/<client_slug>
   Main client portal (renders portal.html)

POST /api/portal/<client_slug>/upload
   Upload CSV file
   Form: {file: File, file_type: 'balance'|'payout'|'bank'}

GET /api/portal/<client_slug>/status
   Get upload status for all files
   Response: {status, balance, payout, bank, exceptions_count}

GET /api/portal/<client_slug>/preview
   Get data preview (row counts, exceptions list)

POST /api/portal/<client_slug>/process
   Generate and download Excel report
   Response: Excel file download

POST /api/portal/<client_slug>/reset
   Clear session and all uploaded files
